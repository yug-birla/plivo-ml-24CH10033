LLM 2,000 Step Speedrun Summary

1. Architecture Overview

We engineered a highly optimized 3-layer Transformer architecture, rigorously staying under the 2,000,000 parameter budget. The model totals exactly 1,935,104 parameters. To maximize learning efficiency in just 2,000 steps, standard transformer components were upgraded with modern, parameter-efficient primitives:

  • SwiGLU Activations: Replaced standard GELU/ReLU FFNs. We increased the hidden dimension to 512, drastically improving the model's capacity to represent multi-lingual structures.
  • Grouped Query Attention (GQA): 8 Query heads and 2 KV heads. This reduced the parameter overhead of the attention mechanism, allowing us to reinvest those parameters into a wider embedding dimension (d_model=256).
  • Rotary Positional Embeddings (RoPE): Replaced learned absolute positional embeddings, freeing up parameters while providing superior sequence length extrapolation.
  • RMSNorm: Used instead of LayerNorm for slightly faster execution and fewer parameters.

2. Key Optimizations for Bits-Per-Byte (BPB)

BPE Tokenization (Vocab 1024): The most critical bottleneck was the representation of Devanagari (Hindi) text. A byte-level tokenizer fragments Devanagari characters into 3 bytes each, destroying the effective context window. We built a pure-Python Byte-Pair Encoding (BPE) tokenizer that compresses Devanagari natively. This reduces Hindi sequence lengths by ~3x, allowing the model to learn long-range dependencies within a smaller block size (256).

  • Weight Tying: The token embedding matrix and the final linear projection head share weights. This recovered exactly 262,144 parameters, allowing us to push the embedding size to 256.
  • Optimization Strategy: AdamW optimizer with a Cosine Annealing learning rate scheduler (Warmup 200, Max LR 3e-3). This aggressive schedule is precisely calibrated to reach peak convergence at the 2,000-step mark.
