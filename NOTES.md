# Architecture Notes

My best configuration utilizes a 3-layer LLaMA-style micro-architecture with exactly 1,935,104 parameters, coupled with a custom pure-Python Byte Pair Encoding (BPE) tokenizer (vocab size 1024). A standard byte-level tokenizer struggles severely with Devanagari text which demands 3 bytes per character, forcing the model to expend parameters just to reconstruct UTF-8; BPE solves this natively by reducing fragmentation and expanding the effective context window. By implementing Grouped Query Attention (GQA), RMSNorm, and weight tying between the embedding and output matrices, I recovered hundreds of thousands of parameters. These recovered parameters were aggressively reinvested into expanding the SwiGLU hidden dimension to 512 and the embedding dimension to 256, dramatically enhancing the model's representational capacity. Finally, Rotary Positional Embeddings (RoPE) provided parameter-free sequence extrapolation, while a Cosine Annealing learning rate schedule ensured optimal loss convergence within the strict 2,000 optimizer step constraint.
