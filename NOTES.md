# Architecture Notes

My best valid checkpoint is `ckpt.pt`, which evaluates at 2.026 BPB on `data/dev_eval.txt` with 1,935,104 parameters and 2,000 recorded optimizer steps. The configuration uses a 3-layer LLaMA-style micro-architecture with a 1024-token byte-fallback BPE tokenizer, 256-token context, 256 embedding width, 8 query heads, 2 KV heads, RMSNorm, RoPE, SwiGLU FFNs, and tied embeddings. A standard byte-level tokenizer struggles with Devanagari text because each character often consumes three byte tokens, so BPE improves effective context length while preserving arbitrary UTF-8 reconstruction. Weight tying recovers output-head parameters that are better spent on width and feed-forward capacity. AdamW with warmup, cosine decay, and gradient clipping was used to improve convergence within the strict 2,000-step budget.
